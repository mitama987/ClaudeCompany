Claude Terminal Shortcuts

Windows Terminal に Claude Code 起動用のショートカットを追加する配布キットです。 既存設定をバックアップし、指定したプロジェクトフォルダから `claude` を起動します。

インストール手順 GitHub Pages 公開

Quick Start

GitHub から導入する

まず利用者の PC で Claude Code の `claude` コマンドが動く状態にします。 次に、このリポジトリの `install.ps1` を実行します。

Raw スクリプト方式

$script = "$env:TEMP\install-claude-terminal-shortcuts.ps1"
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/OWNER/REPO/main/install.ps1" -OutFile $script
powershell -ExecutionPolicy Bypass -File $script -ProjectPath "C:\path\to\project"

Clone 方式

git clone https://github.com/OWNER/REPO.git
cd REPO
powershell -ExecutionPolicy Bypass -File .\install.ps1 -ProjectPath "C:\path\to\project"
  1. 1 `ProjectPath` には Claude Code を開きたいプロジェクトフォルダを指定します。
  2. 2 初回だけ Claude Code の `Yes, I trust this folder` を承認します。
  3. 3 以後は `Ctrl+D`、`Ctrl+E`、`Ctrl+T` から直接 Claude Code を起動できます。

What It Changes

追加されるショートカット

キー 動作 起動先
Ctrl + D 右ペインで Claude Code 指定した `ProjectPath`
Ctrl + E 下ペインで Claude Code 指定した `ProjectPath`
Ctrl + T 新規タブで Claude Code 指定した `ProjectPath`

Safety

配布時の安全ルール

`~/.claude.json` や Claude の認証情報は配布しません。 `--dangerously-skip-permissions` は確認画面だけでなく権限確認も緩めるため、標準手順には入れません。

配るもの

`install.ps1`、`docs/`、README、LICENSE。

配らないもの

`~/.claude.json`、個人の Windows Terminal 設定全体、API キー。

利用者が行うこと

自分の `ProjectPath` を指定し、初回だけ trust ダイアログを承認します。

Troubleshooting

よくある詰まりどころ

まだ trust 画面が出る

`ProjectPath` が実際に開きたいフォルダと一致しているか確認します。別パス表記のフォルダは Claude Code 側で別ワークスペースとして扱われることがあります。

`claude` が見つからない

Claude Code をインストールし、通常の Windows Terminal で `claude --version` が動くことを確認します。`cc` を使いたい場合は `-LaunchCommand cc` を指定できます。

ショートカットが別用途と衝突する

`install.ps1` は対象キーを上書きします。既存設定は `.bak-日時` としてバックアップされるため、必要に応じて復元できます。
