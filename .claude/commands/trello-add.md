# Trello カード追加スキル

ユーザーの指示からカードを作成し、締切日の昇順で正しい位置に配置する。

## 引数

$ARGUMENTS

## ボード情報

- **ボードID**: `5ffe5b3fa17b04881f2c4fbc`
- **ボード名**: データに基づく原因究明

## リスト一覧

| リスト名 | ID | 用途 |
|---------|-----|------|
| 今日（前半）| `5ffe5b3fa17b04881f2c4fbf` | 今日のタスク |
| 今日（後半）| `5ffe5b3fa17b04881f2c4fbe` | **明日の予定**（名前と用途が異なるので注意） |
| 今週（平日）| `5ffe5b3fa17b04881f2c4fc0` | 今週のタスク |
| {当月}月 | `5ffe5b3fa17b04881f2c4fc3` | 当月のタスク（月次処理でリネーム） |
| {翌月}月 | `5ffe5b3fa17b04881f2c4fc4` | 翌月以降のタスク（月次処理でリネーム） |

## Trello API認証

プロジェクトルートの `.env` ファイルから読み込む：

```bash
source .env
# TRELLO_API_KEY, TRELLO_TOKEN が設定される
```

## 実行手順

### Step 1: 引数を解析

ユーザーの入力から以下を特定する：
- **カード名**（必須）
- **締切日**（必須。相対指定「来週火曜」なども今日の日付から計算）
- **対象リスト**（任意。指定がなければ Step 2 で自動判定）
- **説明**（任意）
- **ラベル**（任意）

### Step 2: 対象リストを自動判定（リスト未指定の場合）

締切日と今日の日付を比較して判定する：

1. **今日** → `今日（前半）`
2. **明日** → `今日（後半）`
3. **今週中（今日・明日以外）** → `今週（平日）`
4. **今月中（今週以降）** → 当月リスト（リストID `5ffe5b3fa17b04881f2c4fc3`）
5. **来月以降** → 翌月リスト（リストID `5ffe5b3fa17b04881f2c4fc4`）

※ 月名リストが存在しない場合は `get_lists` で最新のリスト一覧を取得して確認する。

### Step 3: カードを作成

Trello MCP の `add_card_to_list` を使用：
- `listId`: 対象リストのID
- `name`: カード名
- `dueDate`: ISO 8601形式。**JST締切日の前日15:00 UTC** を設定する（= JST当日 00:00）
  - 例: 締切 4/26 JST → `2026-04-25T15:00:00.000Z`
  - 例: 締切 4/29 JST → `2026-04-28T15:00:00.000Z`
  - ⚠️ よくあるミス: 締切日そのものを `YYYY-MM-DDT15:00:00.000Z` にすると JST翌日扱いになり日付が1日ずれる
- `description`: あれば設定

作成されたカードの `id` を控える。

### Step 4: リスト内のカードを取得

Trello MCP の `get_cards_by_list_id` で対象リストの全カードを取得する。

### Step 5: 正しい位置（pos）を計算

**ラベル優先度 + 締切日の複合ソートで位置を決定する。**

#### 5-1: ラベル優先度グループ

| 優先度 | ラベル色 | ID |
|--------|---------|-----|
| 1 | ボールドブルー「メール返信」 | `69d05b900f03e0a77e66f384` |
| 2 | 緑 | `5ffe5b40a17b04881f2c5196` |
| 3 | 黒 | `644b05b30003aaa5d54794c2` |
| 4 | その他（ラベルなし含む） | - |

- 複数ラベル付きカードは最高優先度のラベルで判定

#### 5-2: グループ内で締切日ソート

各ラベルグループ内で:
- **締切ありカード**: `due` 昇順でソート
- **締切なしカード**: グループ内の最後

#### 5-3: 新カードの挿入位置を特定

新カードのラベルと締切日から、ソート済みリスト内の正しい位置を特定する。
前後のカードの `pos` 値から中間値を計算する：
- **先頭に入る場合**: `先頭カードのpos / 2`
- **中間に入る場合**: `(前のpos + 後のpos) / 2`
- **末尾に入る場合**: `最後のpos + 65536`
- **リストが空の場合**: `"top"`

### Step 6: Trello REST API で位置を設定

Bash の curl コマンドで直接APIを叩く：

```bash
source .env
curl -s -X PUT "https://api.trello.com/1/cards/{cardId}?pos={計算したpos}&key=${TRELLO_API_KEY}&token=${TRELLO_TOKEN}"
```

### Step 7: 結果を報告

以下のフォーマットで完了報告する：

```
追加しました！

- カード: {カード名}
- リスト: {リスト名}
- 締切: {MM/DD（曜日）}
- 位置: {前のカード名}の後 / リスト先頭
```

## 注意事項

- 締切日は必ず JST で解釈する。UTC変換は **「締切日-1日」T15:00:00.000Z** （= JST当日0時）
  - 例: 締切 4/26 JST → `2026-04-25T15:00:00.000Z`（✗ `2026-04-26T15:00:00.000Z` は JST 4/27 扱い）
- 同じ締切日のカードが既にある場合は、そのカードの直後に配置
- リスト名を「該当リスト」と言われた場合は、締切月のリストを指す
- ユーザーが明示的にリストを指定した場合はそちらを優先
